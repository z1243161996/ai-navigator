第一步：获取 Cloudflare Account ID

1打开 Cloudflare Dashboard，登录你的账号

  1. 点击左侧菜单 「Workers 和 Pages」
  2. 在页面 右侧边栏 找到 「账户 ID」（Account ID）
  3. 点击旁边的复制按钮，复制这个 32 位字符串
Account ID: a1b2c3d4e5f67890abcdef1234567890（示例）
找不到？ 也可以在：账户首页 → 任意域名 → 右下角「API」区域找到

第二步：创建 Cloudflare API Token

2打开 Cloudflare API Tokens 页面

  1. 或者：右上角头像 → 「我的个人资料」 → 左侧 「API 令牌」
  2. 点击 「创建令牌」（Create Token）
  3. 在模板列表中，找到 「Edit Cloudflare Workers」
  4. 点击右侧 「使用模板」（Use template）

3配置令牌权限

权限范围权限类型说明
Account / Cloudflare PagesEdit允许部署页面（模板已预填）
Account / Workers ScriptsEdit模板已预填
账户资源选择你的账户下拉选择
  1. 确认权限无误，点击 「继续以显示摘要」
  2. 点击 「创建令牌」
重要：Token 只显示一次！ 创建后立即复制保存。如果丢失只能重新创建。
API Token: Abc123Xyz456_qwerty789...（示例，实际更长）

第三步：在 GitHub 添加 Secrets

你已在此页面：Settings → Secrets and variables → Actions

4添加 CLOUDFLARE_ACCOUNT_ID

  1. 点击绿色按钮 「New repository secret」
  2. Name 输入：CLOUDFLARE_ACCOUNT_ID
  3. Secret 粘贴第一步复制的 Account ID
  4. 点击 「Add secret」

5添加 CLOUDFLARE_API_TOKEN

  1. 再次点击 「New repository secret」
  2. Name 输入：CLOUDFLARE_API_TOKEN
  3. Secret 粘贴第二步创建的 API Token
  4. 点击 「Add secret」
添加完成后页面应显示：

Repository secrets
├── CLOUDFLARE_ACCOUNT_ID    Updated just now
└── CLOUDFLARE_API_TOKEN     Updated just now

第四步：触发部署

6在终端执行以下命令触发 CI/CD：

git commit --allow-empty -m "ci: trigger deploy"
git push

或者在任何代码变更后 git push 即可自动触发。

第五步：验证部署

7确认部署成功

  1. 打开仓库 Actions 标签页，查看 workflow 运行状态
  2. 绿色 ✓ 表示部署成功
  3. 打开 Cloudflare Dashboard → Workers & Pages
  4. 找到 ai-navigator 项目，查看部署 URL
默认域名：ai-navigator.pages.dev

自定义域名（可选）

  1. Cloudflare Dashboard → Workers & Pages → ai-navigator
  2. 点击 「自定义域」 标签
  3. 点击 「设置自定义域」
  4. 输入你的域名（如 nav.yourdomain.com）
  5. 如果域名在 Cloudflare 管理，DNS 记录会自动添加

常见问题

问题解决方案
Authentication error检查 CLOUDFLARE_API_TOKEN 是否正确，重新创建
Account not found检查 CLOUDFLARE_ACCOUNT_ID 是否正确
npm ci failed确保 web/package-lock.json 已提交到仓库
页面 404检查 build output 是 web/dist，确认 _redirects 文件存在
